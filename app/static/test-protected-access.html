Test Protected Access

This page demonstrates how to use the password-protection.html page for protecting file access with a password.

Example Protected URL

The following link points to an example file that is protected with the password testpassword:

password-protection.html#aHR0cHM6Ly9kdW1teS1maWxlLmNvbS9zYW1wbGUucGRm:test-id:testpassword

Note: The URL format is:

password-protection.html#[encoded-file-url]:[share-id]:[password]

Where:

  • [encoded-file-url] is the Base64-encoded URL of the file
  • [share-id] is a unique identifier for the share
  • [password] is the password to access the file

How It Works

  1. When a user clicks on the protected URL, they are taken to the password-protection.html page.
  2. The page extracts the encoded file URL, share ID, and password from the hash fragment in the URL.
  3. The user is prompted to enter the password.
  4. If the password matches, the user is redirected to the actual file URL.
  5. If the password doesn't match, an error message is displayed.

This mechanism works entirely client-side and doesn't require server-side processing for the password verification.
